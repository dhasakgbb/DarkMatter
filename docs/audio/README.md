# Photon Audio Documentation

Photon has two audio paths: procedural synthesis for the default playable game and manifest-driven runtime assets for director-temp, A/B, and final studio material. This directory is the canonical handoff point for audio direction, ingest, verification, and procurement.

## Current Runtime Truth

| Area | Current State |
|---|---|
| Default audible path | `settings.proceduralAudio` starts `true`; `ProceduralSynth` generates engine, drones, UI, and SFX in Web Audio. |
| Asset path | `photon/src/audio.ts` loads enabled entries from `photon/src/audio-manifest.json`. Bundled assets under `photon/src/audio-assets/` are inlined by Vite; external dev/server assets can live under `photon/public/audio/`. |
| Checked-in assets | The current `.ogg` pack is deterministic director-temp audio generated by `tools/generate_director_audio.py`, not final studio audio. |
| Final audio status | Final commissioned or separately licensed music/SFX have not landed in the manifest as production assets. |
| Trace/debug hook | `window.__PHOTON_AUDIO_TRACE` records asset, music, and missing asset events in asset mode. `window.render_game_to_text()` reports audio mode, readiness, engine state, and drone state. |

## Common Commands

Run from the repository root unless the command changes directory explicitly.

| Task | Command |
|---|---|
| Audit manifest paths and OGG decode health | `node tools/audit_audio_manifest.mjs` |
| Emit machine-readable audit JSON | `node tools/audit_audio_manifest.mjs --json` |
| Regenerate director-temp assets | `python3 tools/generate_director_audio.py` |
| Run audio/unit coverage | `cd photon && npm test` |
| Run typecheck | `cd photon && npm run typecheck` |
| Build runtime bundle | `cd photon && npm run build` |
| Run fast QA gate | `cd photon && npm run qa:fast` |
| Browser audio smoke | `node tools/playtest_audio_cdp.mjs` |
