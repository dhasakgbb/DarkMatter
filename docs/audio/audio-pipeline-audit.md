## Scope

This audit covers:

- Runtime audio orchestration in `photon/src/audio.ts`.
- Manifest shape and current checked-in assets in `photon/src/audio-manifest.json` and `photon/src/audio-assets/`.
- External runtime drop-zone behavior in `photon/public/audio/`.
- Gameplay/UI call sites that trigger audio cues.
- Test, build, and browser-smoke paths that prove the pipeline still works.

This audit does not certify final artistic quality, final mix, commercial licensing for future third-party assets, or platform-store compliance. Those require final asset delivery, license review, and listening tests.

## Runtime Architecture

Photon exposes a single facade, `audio`, from `photon/src/audio.ts`. Gameplay code calls this facade; the facade can only play decoded manifest assets.

```text
game/ui/hazards/photon/memories/witness
  -> audio facade in photon/src/audio.ts
    -> decoded buffers from photon/src/audio-manifest.json
      -> AudioContext master gain
        -> browser audio destination
```

Important runtime rules:

- `startRun()` calls `audio.ensure()`, `audio.resume()`, and `audio.startEngine()` before settings are applied for the run.
- `audio.ensure()` lazily creates the `AudioContext`, master gain, and async manifest asset load.
- The master gain starts at `0.7`; user volume and mute apply through `settings.masterVol` and `settings.muted` after the audio context exists.
- The pause menu has no audio-backend toggle. The runtime mode is always `asset`.
- Asset URLs resolve first against bundled `photon/src/audio-assets/**/*.ogg` via Vite `import.meta.glob`; missing bundled paths fall back to `new URL(path, new URL('audio/', document.baseURI))`, which maps to `photon/public/audio/` in dev/server builds.
- Missing or undecodable files stay silent and are recorded in `window.__PHOTON_AUDIO_TRACE` with `source: 'missing'`.
- Successful SFX and music starts are traced with `source: 'asset'` or `source: 'music'`.
- `window.__PHOTON_AUDIO_TRACE` keeps the latest 400 trace entries.
- `render_game_to_text()` reports `audio.mode`, `audio.assetsReady`, `audio.engineActive`, and `audio.droneActive` for automated playtests.

## Asset Pipeline

### Runtime Asset Lanes

| Lane | Path | Use | Build Behavior |
|---|---|---|---|
| Bundled runtime assets | `photon/src/audio-assets/` | Small OGG files that must work in the single-file build. | Vite inlines them through `import.meta.glob`. |
| External runtime assets | `photon/public/audio/` | Larger dev/server assets or externally hosted assets. | Resolved relative to browser `audio/`; not automatically inlined into the standalone file. |
| Source masters | Outside runtime folders or private source archive | WAV masters, invoices, license PDFs, vendor notes. | Should not be shipped blindly in the browser bundle. |
| Manifest contract | `photon/src/audio-manifest.json` | Declares music stems, SFX variants, gains, rates, loops, spatial flags, and enablement. | Loaded by `photon/src/audio.ts` at runtime. |

### Ingest Runbook

1. Confirm license scope before import: commercial game distribution, trailers, storefront media, streams, social clips, attribution, Content ID, and AI/ML restrictions.
2. Keep invoices, source licenses, and WAV masters outside public runtime folders unless there is an explicit reason to ship them.
3. Export runtime copies as OGG, preferably from 48 kHz source masters.
4. Use lowercase kebab-case folders and filenames that match the existing manifest style.
5. Put small standalone-required runtime files under `photon/src/audio-assets/`.
6. Put large dev/server assets under `photon/public/audio/` only when standalone inlining is not required.
7. Update `photon/src/audio-manifest.json` with cue paths, variants, gain, rate, loop, and spatial flags.
8. Leave new entries `enabled: false` until the file exists and licensing is verified. Generated reference assets are the current exception and are covered by `photon/public/audio/TEMP_LICENSE.md`.
9. Run `node tools/audit_audio_manifest.mjs` from the repository root.
10. Run `cd photon && npm test`, `cd photon && npm run typecheck`, and `cd photon && npm run build`.
11. Playtest the asset-only runtime from the title screen through death/retry and at least one epoch transition.

## Acceptance Gate For Final Audio

Before calling an audio delivery final, verify all of these in a browser build, not only through unit tests:

- `render_game_to_text().audio.mode` reports `asset`.
- Every enabled manifest path exists and decodes through `node tools/audit_audio_manifest.mjs`.
- No archived or generated-code audio path is callable from runtime source.
- At least one SFX cue and one epoch stem set have been verified by ear from the asset path.
- Epoch transitions fade old music and start the new epoch material without abrupt cuts.
- Common SFX do not stack into clipping during high-speed play.
- Spatial hazard whooshes remain readable without masking line gates, damage, or death.
- Heat Death thins intentionally and does not play ordinary race clutter.
- The final asset license packet is complete before marking production audio as shippable.

## Troubleshooting

| Symptom | Likely Cause | Verify | Fix |
|---|---|---|---|
| No audio after page load | Browser audio context has not been created/resumed by a user gesture. | Start a run, then inspect `render_game_to_text().audio.engineActive`. | Ensure gameplay start calls `audio.ensure()` and `audio.resume()`; interact with the page before expecting sound. |
| No asset sound | Assets are still loading, missing, disabled, or not decoded. | Check `render_game_to_text().audio.assetsReady` and `window.__PHOTON_AUDIO_TRACE`. | Run `node tools/audit_audio_manifest.mjs`; enable only verified entries; confirm paths exist in `src/audio-assets` or `public/audio`. |
| Standalone build has no new asset | File was placed under `photon/public/audio/`, which is not the inlined single-file lane. | Run the standalone build and inspect whether the asset URL is external. | Move small required files to `photon/src/audio-assets/` and update the manifest path. |
| Missing cue stays silent | Runtime intentionally returns `false` for missing or unavailable buffers. | Inspect `window.__PHOTON_AUDIO_TRACE` for `source: 'missing'`. | Add the file, fix the manifest path, or leave the cue disabled until ready. |
| Redshift/flow feel static | These signals are currently reserved for future manifest-stem automation. | Compare late-epoch and high-flow play against intended music direction. | Add stem gain/filter controls or dedicated adaptive layers before final mix signoff. |
| Heat Death lacks final identity | Current Heat Death files are 360-second generated reference beds, not commissioned production audio. | Run `node tools/audit_audio_manifest.mjs --json` and `node tools/score_audio_engagement.mjs --write-report`. | Replace with commissioned long-form or modular Heat Death material before final signoff. |
| Browser smoke cannot launch | Chrome path or CDP ports differ on the machine. | Check `PHOTON_PLAYTEST_PORT`, `PHOTON_CHROME_PORT`, and `chromeBin` in `tools/playtest_audio_cdp.mjs`. | Set the ports via environment variables or adjust the local Chrome path. |

## Risks And Open Work

| Risk | Impact | Current Mitigation | Recommended Next Step |
|---|---|---|---|
| Generated reference assets are enabled in the manifest | Maintainers may mistake placeholder audio for final production assets. | Docs and manifest notes identify them as generated temp audio. | Replace with commissioned/licensed assets and update license records. |
| Redshift/flow have no current audible automation | Final asset path may feel flatter than the design target. | Runtime exposes `setRedshift` and `setFlow` as inert extension points. | Decide whether final music uses static epoch stems or adds adaptive stem gain/filter controls. |
| Very early cues before `assetsReady` may be silent | First interaction SFX can be missed while decoding finishes. | Engine/music pending states retry after manifest load. | Queue critical one-shot cues only if final UX requires it. |
| Final license packet is not represented in repo | Production readiness cannot be proven from code alone. | Ingest checklist requires private license storage and manifest disablement until verified. | Create a private `AUDIO_LICENSES.md` or equivalent delivery record when final assets arrive. |
| Browser tests do not prove mix quality | Unit tests verify routing contracts, not loudness, masking, or feel. | `tools/playtest_audio_cdp.mjs` exercises runtime traces. | Add a human listening checklist for each final-audio drop. |
