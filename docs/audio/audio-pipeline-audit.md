## Scope

This audit covers:

- Runtime audio orchestration in `photon/src/audio.ts`.
- Procedural synthesis in `photon/src/procedural.ts`.
- Manifest shape and current checked-in assets in `photon/src/audio-manifest.json` and `photon/src/audio-assets/`.
- External runtime drop-zone behavior in `photon/public/audio/`.
- Gameplay/UI call sites that trigger audio cues.
- Test, build, and browser-smoke paths that prove the pipeline still works.

This audit does not certify final artistic quality, final mix, commercial licensing for future third-party assets, or platform-store compliance. Those require final asset delivery, license review, and listening tests.

## Runtime Architecture

Photon exposes a single facade, `audio`, from `photon/src/audio.ts`. Gameplay code calls this facade without caring whether the current backend is procedural or asset-based.

```text
game/ui/hazards/photon/memories/witness
  -> audio facade in photon/src/audio.ts
    -> procedural mode: ProceduralSynth in photon/src/procedural.ts
    -> asset mode: decoded buffers from photon/src/audio-manifest.json
      -> AudioContext master gain
        -> browser audio destination
```

Important runtime rules:

- `startRun()` calls `audio.ensure()`, `audio.resume()`, and `audio.startEngine()` before settings are applied for the run.
- `audio.ensure()` lazily creates the `AudioContext`, master gain, `ProceduralSynth`, and async manifest asset load.
- The master gain starts at `0.7`; user volume and mute apply through `settings.masterVol` and `settings.muted` after the audio context exists.
- Procedural mode is the default. Asset mode is selected by the pause-menu `Procedural audio` toggle, which flips `settings.proceduralAudio` and calls `applySettings()`.
- Switching to procedural mode fades/stops studio music and asset engine nodes, then restarts the procedural drone and engine if they were pending.
- Switching to asset mode stops the procedural drone/engine and starts decoded music/engine assets if available.
- Asset URLs resolve first against bundled `photon/src/audio-assets/**/*.ogg` via Vite `import.meta.glob`; missing bundled paths fall back to `new URL(path, new URL('audio/', document.baseURI))`, which maps to `photon/public/audio/` in dev/server builds.
- Missing or undecodable asset-mode files stay silent and are recorded in `window.__PHOTON_AUDIO_TRACE` with `source: 'missing'`.
- Successful asset SFX and music starts are traced with `source: 'asset'` or `source: 'music'`.
- `window.__PHOTON_AUDIO_TRACE` keeps the latest 400 trace entries.
- `render_game_to_text()` reports `audio.mode`, `audio.assetsReady`, `audio.engineActive`, and `audio.droneActive` for automated playtests.

## Cue Map

| Cue or Signal | Trigger Surface | Procedural Path | Asset/Manifest Path | Notes |
|---|---|---|---|---|
| Engine loop | `startRun()`, `audio.startEngine()`, `audio.updateEngine()` | Noise bed plus boost oscillator | `sfx.engineLoop` | Speed and boost continuously modulate playback/filter/gain in both modes. |
| Epoch drone/music | `setEpoch()` for non-Heat Death epochs | `ProceduralSynth.startDrone(epoch)` | `music.<epoch>` stems | Asset mode starts all enabled stems for the epoch and fades them in. |
| Heat Death | `setEpoch()` when `epoch.isHeatDeath` | 360-second low-tone fade | `music.Heat Death` stems | Current asset temp material is 32-second low tone/texture, so final Heat Death assets still need long-form delivery. |
| Redshift | `setSkyRedshift()` and late-epoch ticks | Retunes drone pitch voices and softens the drone | No direct asset equivalent | Asset mode does not currently automate music stem pitch/filter from redshift. |
| Flow intensity | Main loop `audio.setFlow(game.flowLevel)` | Opens drone filter and raises wet send | No direct asset equivalent | This is one of the strongest procedural-only adaptive features. |
| Pickup | `Photon.collect()` and pickup hazards | `pickup()` | `sfx.pickup` | Common cue, 3 asset variants. |
| Speed pad | Racing line speed pads and gravity sling rewards | `speedPad()` | `sfx.speedPad` | Also used by gravity-well near-miss rewards. |
| Line gate | Racing gate hits and every third phase streak | `lineGate(streak)` | `sfx.lineGate` | Procedural and asset modes lift pitch/rate with streak. |
| Gate miss | Missed racing gate after a nonzero streak | `gateMiss()` | `sfx.gateMiss` | Negative read when a clean line breaks. |
| Rail scrape | Playfield edge strain and clamp | `railScrape()` | `sfx.railScrape` | Runtime cooldown prevents excessive stacking. |
| Wavelength shift | `Photon.shift(idx)` | `shift(idx)` | `sfx.wavelengthShift` | Gamma, visible, and radio variants are rate-tinted. |
| Damage hit | `Photon.hit()` | `hit()` | `sfx.damageHit` | Also pairs with visual trauma/hit-stop. |
| Death | `beginDeath()` | `death()` | `sfx.death` | Drone and engine stop after the death cue starts. |
| Witness chime | `triggerWitness()` | `witnessChime()` | `sfx.witnessChime` | Plays after Heat Death witness trigger, with engine/drone stopped. |
| Memory unlock | `unlockMemory()` | `memoryUnlock()` | `sfx.memoryUnlock` | Asset/procedural path has a cooldown in asset mode. |
| Hazard whoosh | Hazard pass-by crossing the photon | `whoosh(..., 'generic')` | `sfx.hazardWhoosh` | Asset mode uses HRTF panner when position is supplied. |
| Gravity-well whoosh | Gravity well pass-by | `whoosh(..., 'well')` | `sfx.gravityWellWhoosh` | Heavier spatial pass-by, longer cooldown. |
| UI tick/click | Global button hover/click listeners | `uiTick()`, `uiClick()` | `sfx.uiTick`, `sfx.uiClick` | Applies to all buttons. |
| UI swoosh | Overlay state changes that are not run/pause transitions | `uiSwoosh()` | `sfx.uiSwoosh` | Keeps gameplay transitions from sounding like menu chrome. |
| Epoch riser | Epoch transitions after the first epoch in a run | `epochRiser()` | `sfx.epochRiser` | Not played on first epoch setup. |

## Asset Pipeline

### Runtime Asset Lanes

| Lane | Path | Use | Build Behavior |
|---|---|---|---|
| Bundled runtime assets | `photon/src/audio-assets/` | Small OGG files that must work in the single-file build. | Vite inlines them through `import.meta.glob`. |
| External runtime assets | `photon/public/audio/` | Larger dev/server assets or externally hosted assets. | Resolved relative to browser `audio/`; not automatically inlined into the standalone file. |
| Source masters | Outside runtime folders or private source archive | WAV masters, invoices, license PDFs, vendor notes. | Should not be shipped blindly in the browser bundle. |
| Manifest contract | `photon/src/audio-manifest.json` | Declares music stems, SFX variants, gains, rates, loops, spatial flags, and enablement. | Loaded by `photon/src/audio.ts` at runtime. |

### Ingest Runbook

1. Confirm license scope before import: commercial game distribution, trailers, storefront media, streams, social clips, attribution, Content ID, and AI/ML restrictions.
2. Keep invoices, source licenses, and WAV masters outside public runtime folders unless there is an explicit reason to ship them.
3. Export runtime copies as OGG, preferably from 48 kHz source masters.
4. Use lowercase kebab-case folders and filenames that match the existing manifest style.
5. Put small standalone-required runtime files under `photon/src/audio-assets/`.
6. Put large dev/server assets under `photon/public/audio/` only when standalone inlining is not required.
7. Update `photon/src/audio-manifest.json` with cue paths, variants, gain, rate, loop, and spatial flags.
8. Leave new entries `enabled: false` until the file exists and licensing is verified. Director-temp generated assets are the current exception and are covered by `photon/public/audio/TEMP_LICENSE.md`.
9. Run `node tools/audit_audio_manifest.mjs` from the repository root.
10. Run `cd photon && npm test`, `cd photon && npm run typecheck`, and `cd photon && npm run build`.
11. Playtest both procedural and asset modes from the pause-menu toggle.

## Acceptance Gate For Final Audio

Before calling an audio delivery final, verify all of these in a browser build, not only through unit tests:

- Procedural mode still boots by default and remains audible without any manifest assets.
- Asset mode can be toggled from pause and reports `audio.mode: "asset"` through `render_game_to_text()`.
- Toggling back reports `audio.mode: "procedural"` and restores engine/drone activity.
- Every enabled manifest path exists and decodes through `node tools/audit_audio_manifest.mjs`.
- At least one SFX cue and one epoch stem set have been verified by ear from the asset path.
- Epoch transitions fade old music and start the new epoch material without abrupt cuts.
- Common SFX do not stack into clipping during high-speed play.
- Spatial hazard whooshes remain readable without masking line gates, damage, or death.
- Heat Death thins intentionally and does not play ordinary race clutter.
- The final asset license packet is complete before marking production audio as shippable.

## Troubleshooting

| Symptom | Likely Cause | Verify | Fix |
|---|---|---|---|
| No audio after page load | Browser audio context has not been created/resumed by a user gesture. | Start a run, then inspect `render_game_to_text().audio.engineActive`. | Ensure gameplay start calls `audio.ensure()` and `audio.resume()`; interact with the page before expecting sound. |
| Procedural toggle off but no asset sound | Assets are still loading, missing, disabled, or not decoded. | Check `render_game_to_text().audio.assetsReady` and `window.__PHOTON_AUDIO_TRACE`. | Run `node tools/audit_audio_manifest.mjs`; enable only verified entries; confirm paths exist in `src/audio-assets` or `public/audio`. |
| Standalone build has no new asset | File was placed under `photon/public/audio/`, which is not the inlined single-file lane. | Run the standalone build and inspect whether the asset URL is external. | Move small required files to `photon/src/audio-assets/` and update the manifest path. |
| Missing cue stays silent | Asset mode intentionally returns `false` for missing or unavailable buffers. | Inspect `window.__PHOTON_AUDIO_TRACE` for `source: 'missing'`. | Add the file, fix the manifest path, or leave the cue disabled until ready. |
| Asset mode feels less alive than procedural mode | Redshift and flow intensity currently modulate only the procedural drone. | Compare asset mode to procedural mode during late epochs or high-flow play. | Add asset stem automation or dedicated adaptive layers before final mix signoff. |
| Heat Death asset mode ends too soon | Current director-temp Heat Death files are 32-second test assets. | Run `node tools/audit_audio_manifest.mjs --json` and inspect durations. | Replace with commissioned long-form or modular Heat Death material. |
| Browser smoke cannot launch | Chrome path or CDP ports differ on the machine. | Check `PHOTON_PLAYTEST_PORT`, `PHOTON_CHROME_PORT`, and `chromeBin` in `tools/playtest_audio_cdp.mjs`. | Set the ports via environment variables or adjust the local Chrome path. |

## Risks And Open Work

| Risk | Impact | Current Mitigation | Recommended Next Step |
|---|---|---|---|
| Director-temp assets are enabled in the manifest | Maintainers may mistake placeholder audio for final production assets. | Docs and manifest notes identify them as generated temp audio. | Replace with commissioned/licensed assets and update license records. |
| Asset mode lacks procedural redshift/flow adaptation | Final asset mode may feel flatter than the default procedural runtime. | Procedural mode remains default and fully adaptive. | Decide whether final music uses static epoch stems or adds adaptive stem gain/filter controls. |
| Asset mode drops early SFX before `assetsReady` | Very early asset-mode cues may be silent. | Procedural default avoids this during normal play. | Queue critical asset-mode cues only if final UX requires it. |
| Final license packet is not represented in repo | Production readiness cannot be proven from code alone. | Ingest checklist requires private license storage and manifest disablement until verified. | Create a private `AUDIO_LICENSES.md` or equivalent delivery record when final assets arrive. |
| Browser tests do not prove mix quality | Unit tests verify routing contracts, not loudness, masking, or feel. | `tools/playtest_audio_cdp.mjs` exercises runtime traces. | Add a human listening checklist for each final-audio drop. |
